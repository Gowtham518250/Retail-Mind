═══════════════════════════════════════════════════════════════════════════════
  RENDER DEPLOYMENT STEPS
═══════════════════════════════════════════════════════════════════════════════

1. GO TO RENDER DASHBOARD
   - Log in to https://render.com
   - Select your backend service
   - Go to Settings → Environment

2. ADD ENVIRONMENT VARIABLES (Click "Add Environment Variable")
   
   REQUIRED VARIABLES:
   
   Name: DATABASE_URL
   Value: postgresql://user:password@your-host.render.com/dbname?sslmode=require
   (Get this from Render PostgreSQL dashboard)
   
   Name: SECRET_KEY
   Value: (Generate with: python -c "import secrets; print(secrets.token_urlsafe(32))")
   
   Name: SENDER_EMAIL
   Value: your-email@gmail.com
   
   Name: SENDER_PASSWORD
   Value: your-app-password (NOT your Gmail password, use App Password)
   (Get from: https://myaccount.google.com/apppasswords)
   
   Name: SMTP_SERVER
   Value: smtp.gmail.com
   
   Name: SMTP_PORT
   Value: 587
   
   Name: REDIS_URL
   Value: redis://default:password@redis-host:6379
   (Optional - only if using Redis)

3. SAVE & DEPLOY
   - All variables are now set as environment variables
   - No credentials in code or git
   - Deploy your code normally
